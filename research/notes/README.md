# Notes Directory

This directory contains research notes, documentation, and ongoing work.

## Subdirectories

- **research_logs/** - Daily or session-based research activity logs
- **hypotheses/** - Hypotheses and theories being investigated
- **findings/** - Preliminary findings and observations
- **conclusions/** - Final conclusions and summaries
- **questions/** - Outstanding questions and areas for further research

## Note-Taking Guidelines

### Research Logs

Document each research session:
```markdown
## Research Log - YYYY-MM-DD

**Researcher:** [Name]
**Duration:** [Time spent]
**Focus Areas:** [Topics investigated]

### Activities Completed
- [Activity 1]
- [Activity 2]

### Key Findings
- [Finding 1]
- [Finding 2]

### Sources Consulted
- [Source 1]
- [Source 2]

### Questions Raised
- [Question 1]
- [Question 2]

### Next Steps
- [Action item 1]
- [Action item 2]
```

### Hypotheses Documentation

Track hypotheses being tested:
```markdown
# Hypothesis: [Title]

## Statement
[Clear statement of the hypothesis]

## Rationale
[Why this hypothesis is being considered]

## Evidence Supporting
[What evidence supports this hypothesis]

## Evidence Contradicting
[What evidence contradicts this hypothesis]

## Testing Methods
[How this hypothesis is being tested]

## Status
[Proposed/Testing/Confirmed/Rejected]

## Last Updated
[Date]
```

### Findings Documentation

Document preliminary findings:
```markdown
# Finding: [Title]

## Observation
[What was observed]

## Context
[Background and circumstances]

## Evidence
[Supporting evidence]

## Preliminary Analysis
[Initial interpretation]

## Confidence Level
[High/Medium/Low]

## Further Investigation Needed
[What needs to be verified or explored]

## Date Noted
[Date]
```

### Questions Documentation

Track outstanding questions:
```markdown
# Question: [Title]

## Question
[Clear statement of the question]

## Context
[Why this question is important]

## Potential Approaches
[How this question might be answered]

## Status
[Open/Investigating/Answered]

## Related Research
[Links to relevant research areas]

## Date Raised
[Date]
```

## Organization Principles

- Use clear, descriptive titles
- Date all entries
- Cross-reference related notes
- Update status regularly
- Archive resolved items appropriately

## Review Process

Regular review of notes should:
- Update statuses based on new information
- Move confirmed findings to conclusions
- Archive resolved questions
- Identify patterns across notes
- Generate new research questions
